# STEDI-HUMAN-ANALYTICS
STEDI Human Balance Analytics - Data Lake Project

Tech Stack
Cloud Storage: Amazon S3

Data Processing/ETL: AWS Glue (PySpark)

Data Catalog & Querying: AWS Athena, AWS Glue Data Catalog

Languages: Python, SQL
 Data Lake Architecture
The data lake follows a layered architecture to systematically clean and process the data:

Landing Zone: Raw, unprocessed JSON data generated by the devices, stored directly in S3.

Trusted Zone: Sanitized data where PII constraints and GDPR/privacy boundaries are enforced (e.g., filtering out users who did not consent to research sharing).

Curated Zone: Final, optimized datasets created by joining trusted tables together, specifically structured to feed directly into machine learning pipelines.

 ETL Pipeline (AWS Glue Jobs)
The core processing is handled by 5 distinct AWS Glue (PySpark) jobs:

1. customer_landing_to_trusted.py
Source: customer_landing

Target: customer_trusted

Logic: Filters the raw customer records to only include users who have explicitly agreed to share their data for research (shareWithResearchAsOfDate is not null).

2. accelerometer_landing_to_trusted.py
Source: accelerometer_landing, customer_trusted

Target: accelerometer_trusted

Logic: Joins raw accelerometer data with the trusted customer table. It enforces strict temporal privacy by filtering out any accelerometer readings that were captured prior to the user's explicit consent date.

3. customer_trusted_to_curated.py
Source: customer_trusted, accelerometer_trusted

Target: customer_curated

Logic: Isolates a curated list of customers who are both verified to have consented to research AND possess valid accelerometer reading data.

4. step_trainer_landing_to_trusted.py
Source: step_trainer_landing, customer_curated

Target: step_trainer_trusted

Logic: Cleans the raw step trainer data by strictly filtering it against the fully curated customer list using device serial numbers.

5. machine_learning_curated.py
Source: step_trainer_trusted, accelerometer_trusted

Target: machine_learning_curated

Logic: Performs an inner join between the step trainer and accelerometer data based on exact timestamp matches. Drops the user (email) column to fully anonymize the dataset for the data science team.

 AWS Athena Data Catalog
The following tables were defined in AWS Athena using SQL DDL scripts to allow for direct querying of the S3 folders at each stage of the pipeline:

Landing Tables:

customer_landing (956 rows)

accelerometer_landing (81,273 rows)

step_trainer_landing (28,680 rows)

Trusted Tables:

customer_trusted (482 rows)

accelerometer_trusted (40,981 rows)

step_trainer_trusted (14,460 rows)

Curated Tables:

customer_curated (464 rows)

machine_learning_curated ( 43,681 rows)

 How to Run
Create an S3 bucket and upload the raw JSON files into /customer/landing/, /step_trainer/landing/, and /accelerometer/landing/.

Execute the Landing DDL scripts in AWS Athena to generate the base tables.

Run the AWS Glue PySpark scripts in the order listed above to process the data through the Trusted and Curated zones.

Verify the row counts in Athena using SELECT count(*) statements to ensure data integrity at each step.
